Portfolio Redesign

Introduction

The goal of this project was to design and develop a website that informs users about the assistance zoos provide to global conservation work. In doing so, the website also needed to address the misconception that animals in zoos are unhappy or mistreated.

Architecture

While a personal portfolio is a relatively small web project, I wanted to make sure my design will withstand several changes and updates over its lifespan. This need made intuitive organization a necessity. Fortunately, I'm a geek for naming conventions, and rebuilding the website gave me the perfect opportunity to integrate some better structure.

GitHub Organization

When this project was just a handful of HTML files, having a flat organizational structure wasn't really a problem. The update, however, raised a number of new issues with this system. Where should pages no longer in use go? How should images and icons be distinguished from each other? Suddenly hierarchy made a lot of sense.

CSS Frameworks

Another major change in this build was the introduction of a responsive grid framework, affectionately nicknamed "Discount Bootstrap." It was pretty clear that keeping this set of styles in with the rest of the page CSS would just make everything hard to find, so the responsive grid framework was given its own CSS file. With files organized better on GitHub, CSS could be sorted into three main files: main.css, discount-bootstrap.css, and UI.css.
